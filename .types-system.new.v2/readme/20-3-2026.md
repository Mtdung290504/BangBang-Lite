# Note 20-3-2026

## Thống nhất mọi điều chỉnh stat thành 1 type duy nhất

- `DealtDamage`, `RecoverHP`, `ModifyEnergy`, buff/debuff tốc chạy, tốc đánh... **TẤT CẢ** đều là điều chỉnh 1 stat nào đó.
- Khác biệt duy nhất: có loại áp dụng **lập tức** (damage, heal) và loại có **duration** (buff/debuff) → chỉ cần cờ `instant`.
- Không tách type riêng — DSL đập bẹp dát mỏng hết cỡ.

## Text visual do engine tự quyết

- Bỏ `TextVisual`, `text-delta-angle` khỏi DSL.
- Engine tự quyết dựa trên giá trị dương (xanh/hồi) hay âm (đỏ/damage), tự rẽ góc khi nhiều luồng.

## Reduction/mitigation là function pipeline, không phải string enum

- `DamageReductionPolicy` string (`'bypass-armor' | 'use-armor' | 'use-shield'`) quá giới hạn.
- Thực tế: giảm tốc dựa trên kháng hiệu ứng, sát thương giảm bởi giáp + miễn thương + giảm damage skill + ... — logic phức tạp, mỗi stat có cơ chế riêng.
- → Dùng `function[]` (reduction pipeline): engine chạy qua từng function để tính giá trị sau cùng.
- Designer không cần viết function reduction, dùng template có sẵn.

## Không thêm stat đặc biệt kiểu `ally-nearby`, `enemy-nearby`

- Đếm ally/enemy xung quanh dùng **sensor entity pattern** đã có (collider + on-impact + apply effect).
- Đừng ô nhiễm `RuntimeStats` bằng thứ mà hệ thống hiện tại đã giải quyết được.

## Tái sử dụng qua template function, không qua abstract type

- Viết lặp lại khi biểu diễn DSL là không tránh khỏi (các tầng effect stack có thể khác nhau).
- Tái sử dụng = module chứa **template function** cho designer dùng, không phải cố abstract ở tầng type.

## Function resolver trong DSL

- Skill khai báo trong JS module → tận dụng toàn bộ sức mạnh JS: function, biến, loop, conditional.
- `ValueResolver = (ctx) => number` thay thế `ValueWithUnit` string expression ở các điểm cần tính toán phức tạp.
- Giữ `ValueWithUnit` cho case đơn giản (designer-friendly).

## Passive skill = SkillEntry auto-trigger, xóa hết passive types

- `EventTriggeredPassive`, `PeriodicPassive`, `PermanentBuffPassive`, `PhasedPassiveSkill` — **XÓA HẾT**.
- Passive chỉ là `SkillTiming & ActionBased` — y hệt active skill, khác ở engine behavior:
    - Auto-trigger khi CD xong (không cần key press)
    - Immune silent (slot passive không bị câm)
- Trigger luôn là on-cooldown. Event subscription nằm trong **effect mà passive tạo ra**, không phải trong passive type.
- Flow: CD xong → auto-trigger → áp effect lên mình → effect chứa `on-event` hooks → event fires → actions → có thể reset CD passive.

## `on-event` thêm vào EffectImpactManifest

- Effect có thể lắng nghe game event (từ `TankEventActorsMap`).
- Thay thế hoàn toàn khái niệm event-triggered passive.
- Ví dụ:
    - Kirito on-hit: passive CD=0 → áp effect vĩnh viễn → `on-event: { 'on-hit-taken': [buff] }`, CD=∞
    - Khiên periodic: passive CD=10s → áp effect tạo khiên → khiên vỡ (`on-end`) reset CD passive → CD xong → trigger lại
    - On-kill buff: passive CD=0 → áp effect → `on-event: { 'on-destroy': [buff tăng tốc 5s] }`, CD=∞
    - Permanent buff: passive CD=0 → áp effect unremovable với `modify-stats`, CD=∞
- Ưu điểm so với interval effect: CD reset được khi sự kiện xảy ra (khiên vỡ → reset CD), interval thì cứ tick đều không quan tâm.

## Tối giản event: 13 → 6

- Bỏ attacker-side HIT events (trước damage): `on-hit-dealt`, `on-skill-dealt`, `on-normal-attack-dealt` — redundant với on-impact.
- Bỏ filtered events: `on-skill-taken`, `on-normal-attack-taken`, `on-skill-taken-damage`, `on-normal-attack-taken-damage` — gom vào `on-hit-taken` / `on-hit-taken-damage`.
- Bỏ `on-activate-skill` — thêm action vào skill trực tiếp.
- **Giữ `on-hit-dealt-damage`** (gom skill + normal-attack) — on-impact không biết damage có thực sự xảy ra (bị shield/invincible chặn).
- Thêm `on-fatal-damage` — engine pre-calculate, fire TRƯỚC khi chết.
- Đổi `TankEventActorsMap` → `TankEvent` union type.
- 6 events: `on-hit-taken`, `on-hit-taken-damage`, `on-hit-dealt-damage`, `on-fatal-damage`, `on-destroy`, `on-destroyed`.

## `{ stack: 'up' | 'down' }` thay `'clear'`/`'stack'` string literal

- `{ stack: 'up' }` = bồi thêm 1 stack (spell shield counting hits).
- `{ stack: 'down' }` = giảm 1 stack.
- Stack 0 → effect hết. Kết hợp với `CleanEffect` by id/tag cho clean rõ ràng.

## Bỏ condition khỏi ImpactAction

- Giá trị/reduction khác nhau theo điều kiện → xử lý TRONG function resolver (1 action).
- Chưa có case cần actions khác bản chất → bỏ condition để gọn, thêm lại khi cần.

## Triết lý: type gọn, khai báo dài

- Mỗi type thêm = engine thêm code path, thêm bug, thêm maintain.
- Khai báo dài xử lý bằng JS: template function, helper, spread, map.
- Complexity budget đổ vào khai báo (1 lần, có template), không phải type system (vĩnh viễn).
